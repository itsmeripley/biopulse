Build phases
01
Create the main broadcast channel
Set up the central Telegram channel. Add admins. Configure pinned welcome message. This becomes the single source of truth for all official communications.
Setup
02
Create the subgroups
Build 5–6 private, independent Telegram supergroups. Each isolated from the others. All admins added to every group with administrator rights. Member capacity capped at 150.
Build
03
Connect broadcast to all subgroups
Configure automatic message forwarding from the broadcast channel into every subgroup. One post by an admin instantly distributes to all groups simultaneously.
Build
04
Configure Rose Bot in each group
Add and configure Rose Bot across all subgroups. Set welcome messages, group rules, anti-spam thresholds, link blocking, captcha screening, and automated warning limits.
Config
05
Member migration
Announce the new structure to existing members. Distribute unique invite links in batches of ~100–150 per group. Monitor intake and close old groups once migration is complete.
Migration
What this delivers
Single source of truth Admins post once. All groups receive the message instantly and consistently.
Contained security Bad actors are isolated within one group and cannot affect the wider community.
Improved member experience Smaller groups mean conversations are readable, followable, and engaging.
Full admin visibility Admins can enter, monitor, and engage in any subgroup at any time.
Scalable infrastructure Additional groups can be added as the community grows beyond 900 members.
Automated moderation Rose Bot handles routine enforcement so admins focus on community, not policing.
